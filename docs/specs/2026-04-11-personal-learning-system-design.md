# Personal Learning Management System

> Design spec — 2026-04-11
> Status: Approved, ready for implementation planning

## Problem

Links pile up in Slack self-DMs and never get processed. YouTube/news consumption (~7.6h/week) is classified as a single "Learning/Research" bucket, hiding the difference between genuine learning, current events, and filler. There's no system connecting learning goals to resources to scheduled time to progress tracking. The result: learning is reactive and accidental, not intentional and scaffolded.

## Solution

A `/learn` skill backed by an Obsidian-first knowledge graph in `40-learning/`. The skill handles the full pipeline: goal setting, resource ingestion, summarization, scaffolding, and progress tracking. It integrates into the existing open/close rhythm so learning gets planned, scheduled, and reviewed alongside work.

## Time Budget

- **15 min/day** — micro-learning (read one article, watch one tutorial, process inbox links)
- **~1.5h/week** — deep dive session on one topic (scheduled during open-week)
- Some weeks more, but this is the baseline.

## Core Interaction: `/learn` Conversation Flow

### `/learn [topic]` — New Learning Goal

Triggers a guided conversation:

1. **Topic intake** — "What specifically interests you about [topic]? Is this about building, evaluating, understanding the architecture, or something else?"

2. **Current knowledge assessment** — Self-assessment conversation, not a quiz. "Where are you on this right now? Have you worked with any related tools/concepts? What felt unclear or limiting?" Goal: place the learner on a spectrum from "heard the term" to "deep experience, want to go further on X."

3. **Learning outcome** — "What do you want to be able to do after learning this?" Options:
   - Evaluate tools/approaches (make decisions)
   - Build something specific
   - Teach someone else
   - Understand the landscape for informed conversations
   - Something else (open-ended)

4. **Time horizon** — "Is this a 1-week sprint, a month of gradual learning, or an ongoing interest?"

After the conversation, the skill:
- Creates/updates the topic dashboard note in `40-learning/[topic].md`
- Adds it to `_learning-goals.md` with priority (based on relevance to role, current projects, existing goals)
- Kicks off background research — uses WebSearch/Firecrawl to find top resources (docs, tutorials, articles), checks `_inbox.md` for related saved links, builds a scaffolded learning path from current level to target outcome
- Presents the draft learning path for review — the builder can reorder, remove, or add items before it's saved

### `/learn` (no args) — Status Check

Show active goals, inbox status, suggest what to work on next.

### `/learn inbox` — Process Raw Links

Summarize unprocessed links from `_inbox.md`, tag them to topics, surface untagged items for manual assignment.

### `/learn scaffold [topic]` — Refresh Learning Path

AI generates or refines the learning path for a topic based on web research, factoring in current level and target outcome.

## Obsidian Data Model

```
40-learning/
  _inbox.md                    <- raw links (scraped from Slack, added manually)
  _learning-goals.md           <- priority-ranked active learning goals
  _weekly-plan.md              <- this week's learning schedule (generated by open-week)

  agentic-harnesses.md         <- topic dashboard
  agentic-harnesses/
    tool-routing.md            <- concept note (spawned during deep dive)
    memory-patterns.md

  leadership-coaching.md
  leadership-coaching/
    active-listening.md
```

### `_inbox.md` — Raw Link Capture

Append-only log. Each entry has a link, timestamp, source, and AI-generated 1-2 sentence summary. Unprocessed = `- [ ]`, processed = `- [x]` with a tag linking to a topic.

```markdown
- [ ] [OpenClaw: An Open Agent Framework](https://example.com/openclaw) — 2026-04-11, from: Slack self-DM
  > Open-source agent harness focused on tool routing and memory management.
  > Tags: #agentic-harnesses

- [x] [NYT: The AI Job Market Shift](https://nytimes.com/...) — 2026-04-09, from: Slack self-DM
  > Analysis of how AI is reshaping mid-career hiring. -> [[leadership-coaching]]
```

### Concept Notes (e.g., `agentic-harnesses/tool-routing.md`)

Atomic notes spawned during deep dive sessions. Linked back to the topic dashboard. Format is flexible — whatever captures the concept clearly. Minimum: a summary paragraph, key takeaways, and links to related concepts.

## Integration Points

### `/open-week` — Learning Section

Added to the weekly planning flow:

1. Read `_learning-goals.md` for active goals
2. Read each active topic dashboard for `next-session` and progress
3. Ask: "What do you want to learn more about this week?" (confirm existing goals or trigger `/learn` conversation for new ones)
4. Schedule the ~1.5h deep dive block into the weekly plan, written to `_weekly-plan.md`
5. Flag stale goals: "You haven't touched [topic] in 3 weeks — still active, or park it?"
6. AI suggestions: based on work patterns (from operating memo), role, personality, and current projects, suggest learning topics the builder hasn't identified ("You're making hiring decisions weekly but have no leadership assessment training — interested?")

### `/open-day` — Daily Micro-Learning + Link Ingestion

1. **Scrape new links** — check Slack self-DMs (or configured source) for URLs sent since last scrape. Fetch title + summary via WebFetch, append to `_inbox.md`. This is the automatic ingestion step — no manual action needed.
2. Read `_weekly-plan.md` for today's micro-learning assignment
3. Read `_inbox.md` for unprocessed links relevant to active goals
4. Schedule a 15-min "Learn" block on the calendar with the specific resource and topic
5. If no plan exists, pick the highest-priority unread item from the inbox

### `/close-day` — Media Classification + Progress Capture

**Media classification upgrade** in Familiar analysis. Replace the single "Learning/Research" bucket with:

| Category | Heuristic | Examples |
|----------|-----------|---------|
| **Learning** | YouTube titles with tutorial/guide/how-to/framework names, documentation sites, course platforms | "Building Agents with LangGraph - Tutorial", Coursera, MDN docs |
| **Current Events** | Major news sites — article pages, not homepages | NYT article pages, CNN long-reads, The Athletic analysis |
| **Entertainment/Filler** | YouTube with movie/show/music/clip/highlights, social media scrolling, Reddit browsing | "Harry Potter scene", "Nuggets highlights", movie trailers |
| **Mixed/Browse** | Fuzzy middle — homepage browsing, news homepages, general YouTube browsing | NYT homepage, YouTube home feed, general Chrome browsing |

Classification uses Chrome window titles from Familiar captures. The heuristic is keyword-based on the title string — not perfect, but clearly distinguishes the ends of the spectrum. News homepage browsing = Mixed. Specific NYT article = Current Events. YouTube tutorial = Learning. Movie clip = Filler.

**Learning progress capture:** If Familiar shows time on a resource from the learning plan (matched by URL or title), mark it as done in the topic dashboard and log it in the progress section.

### `/close-week` — Learning Review

Added to weekly review output:

- Learning plan completion: X of Y items done
- Time spent on genuine learning vs. filler (using new classification)
- Surface: "You spent 4.2h on YouTube this week. 1.1h was learning content, 0.8h was current events, 2.3h was filler."
- Nudge stale goals into the next week's priorities or suggest parking them

### Obsidian Setup — Onboarding Question

For the personal toolkit, add to the onboarding interview:

"How do you capture interesting links you find online? (Slack self-DMs, Apple Notes, browser bookmarks, email to self, a read-later app, or something else?)"

The answer configures where `/learn inbox` and `/open-day` scrape from. Store in builder profile under `learning_capture_method`. For Slack-based capture, the builder's Slack user ID (from `.env`) is used to read self-DMs via the Slack MCP tool.

Create `40-learning/` folder structure with empty `_inbox.md`, `_learning-goals.md`, and `_weekly-plan.md` during vault setup.

## Link Ingestion (Kevin-specific)

For Kevin: scrape Slack self-DMs as part of `/open-day` or `/learn inbox`. Use the Slack MCP tool to read recent self-DMs, extract URLs, fetch page titles and summaries (via Firecrawl or web fetch), and append to `_inbox.md`.

For the personal toolkit: the ingestion source is configurable based on the onboarding answer. Initial implementation supports Slack self-DMs. Other sources (Apple Notes, email) are future work — the `_inbox.md` format is the same regardless of source, so users can manually paste links there too.

## AI-Suggested Learning Topics

During `/open-week`, the skill can suggest learning topics the builder hasn't identified, based on:

- **Operating memo** — gaps between "I Do" and actual time allocation (e.g., heavy on building, light on management training)
- **Work patterns** — recurring meeting topics where the builder seems to lack context (detected from Fathom summaries or daily note themes)
- **Role + personality** — if builder profile says "VP Engineering" but learning goals are all IC coding topics, surface leadership/management learning
- **Project needs** — if a project involves tech the builder hasn't worked with (detected from project notes or carry-overs)

These are suggestions, not assignments. Presented as: "Based on your work this week, you might benefit from learning about [X]. Want to add it?"

## What This System Does NOT Do

- **Auto-schedule learning over other priorities.** Learning blocks are suggested, not forced. The builder always decides.
- **Replace curiosity browsing.** Random YouTube exploration has value. The system classifies it honestly but doesn't shame it.
- **Require perfect link capture.** The inbox is one input. AI-found resources and manual additions work too. If someone never sends themselves links, the system still works — it just relies more on AI research.
- **Grade or test.** Progress is self-reported via the topic dashboard. The system tracks completion of learning path items, not comprehension.
